Leçon 1 : Introduction au HTML

Bienvenue dans le monde du développement web ! Dans cette première leçon, vous allez découvrir ce qu'est le HTML et comment créer votre première page web.

📖 Qu'est-ce que le HTML ?

HTML signifie HyperText Markup Language (Langage de Balisage HyperTexte). C'est le langage standard utilisé pour créer des pages web. Le HTML n'est pas un langage de programmation, c'est un langage de balisage qui permet de structurer le contenu d'une page web.

Imaginez le HTML comme le squelette d'une maison : il définit la structure de base avant d'ajouter la décoration (CSS) et l'électricité (JavaScript).

🏗️ Structure de base d'un document HTML

Chaque page HTML commence par une structure de base. Voici le modèle minimal :

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Ma première page</title>
</head>
<body>
    <h1>Bonjour le monde !</h1>
    <p>Ceci est ma première page HTML.</p>
</body>
</html>
💡 Explication :
  • <!DOCTYPE html> : Version HTML5
  • <html> : Élément racine
  • <head> : Métadonnées
  • <title> : Titre de l'onglet
  • <body> : Contenu visible
✅ Bonnes pratiques :
  • Toujours fermer vos balises
  • Utiliser l'indentation
  • Écrire en minuscules
  • Ajouter des commentaires

Leçon 2 : Texte & Titres

Apprenez à structurer et formater le texte avec les balises HTML.

📑 Les balises de titres

HTML propose 6 niveaux de titres :

<h1>Titre principal</h1>
<h2>Sous-titre niveau 2</h2>
<h3>Sous-titre niveau 3</h3>
<h4>Sous-titre niveau 4</h4>

<em>Italique</em>
<mark>Surligné</mark>
<del>Barré</del>

Leçon 5 : Les Tableaux

Présentez des données structurées avec des tableaux.

📊 Structure d'un tableau

<table>
    <thead>
        <tr>
            <th>Nom</th>
            <th>Âge</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Alice</td>
            <td>25</td>
        </tr>
        <tr>
            <td>Bob</td>
            <td>30</td>
        </tr>
    </tbody>
</table>
💡 Éléments :
  • <table> : Le tableau
  • <thead> : En-tête
  • <tbody> : Corps
  • <tr> : Ligne
  • <th> : Cellule d'en-tête
  • <td> : Cellule de données

🔗 Fusion de cellules

<!-- Fusion horizontale -->
<td colspan="2">Fusionné</td>

<!-- Fusion verticale -->
<td rowspan="2">Fusionné</td>
📝 Exercices
Exercice 1 :

Créez un tableau de notes (Nom, Matière, Note).

Exercice 2 :

Leçon 6 : Les Formulaires

Collectez des données avec des formulaires interactifs.

📝 Structure d'un formulaire

<form action="/submit" method="post">
    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom">
    
    <button type="submit">Envoyer</button>
</form>

🎛️ Types de champs

<!-- Texte simple -->
<input type="text" placeholder="Votre nom">

<!-- Email -->
<input type="email" placeholder="email@exemple.com">

<!-- Mot de passe -->
<input type="password">

<!-- Nombre -->
<input type="number" min="0" max="100">

<!-- Date -->
<input type="date">

<!-- Case à cocher -->
<input type="checkbox" id="accept">
<label for="accept">J'accepte</label>

<!-- Bouton radio -->
<input type="radio" name="genre" value="homme"> Homme
<input type="radio" name="genre" value="femme"> Femme

<!-- Zone de texte -->
<textarea rows="4" cols="50"></textarea>

<!-- Liste déroulante -->
<select name="pays">
    <option value="fr">France</option>
    <option value="be">Belgique</option>
    <option value="ch">Suisse</option>
</select>

✅ Validation

<input type="text" required>
<input type="email" required>
<input type="text" minlength="3" maxlength="20">
<input type="number" min="18" max="99">
💡 Exemple complet :
<form>
    <label for="nom">Nom :</label>
    <input type="text" id="nom" required>
    
    <label for="email">Email :</label>
    <input type="email" id="email" required>
    
    <label for="message">Message :</label>
    <textarea id="message" rows="5"></textarea>
    
    <button type="submit">Envoyer</button>
</form>
✅ Bonnes pratiques :
  • Toujours associer un label à chaque input
  • Utiliser l'attribut "required" pour les champs obligatoires
  • Choisir le bon type d'input pour chaque donnée
  • Ajouter des placeholders pour guider l'utilisateur
